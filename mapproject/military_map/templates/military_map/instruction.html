คำแนะนำการใช้งาน

สืบเนื่องจาก Pain point ในการฝึกของทหารราบ ที่หน่วยปฏิบัติ และผู้บังคับหน่วย ไม่อาจทราบที่อยู่ที่แน่ชัดของกำลังพล และหน่วยข้างเคียง สามารถทราบได้โดยการใช้วิทยุเพื่อสื่อสารเป็นหลัก ร.29 พัน.1 เห็นถึงปัญหาในข้อนี้ จึงได้ระดมความคิดกันเพื่อพัฒนา Map Command โดยทางหน่วย ได้รับแรงบันดาลใจจากเกม FPS อย่างเช่น CounterStrike และเกม RTS อย่าง Red Alert 2, Warcraft 3 ที่ผู้เล่นสามารถเห็นดำแหน่งของผู้เล่นฝ่ายเดียวกันได้ ทำให้ผู้เล่นสามารถดำเนินกลยุทธ์ที่มีความซับซ้อน ตัดสินใจได้เฉียบคมขึ้น

โดย Map Command เป็นนวัตกรรมหน่วย ร.29 พัน.1 ที่มี Feature หลักๆดังนี้

  • ใช้สัญญานโทรศัพท์มือถือ เพื่อระบุตำแหน่งผู้ใช้งานทุกคนที่อยู่ในแชนแนลเดียวกัน แบบ Real Time
  • ผู้ใช้งานสามารถที่จะวางแผนอย่างง่ายบนแผนที่ (การกำหนดแนวขั้น กำหนดพื้นที่ กำหนดจุดเช็คพ้อยต์ กำหนดเส้นทางการเคลื่อนที่) และผู้ใช้คนอื่น ที่อยู่ในแชนแนลเดียวกัน เห็นภาพที่วาดไว้ไปพร้อมๆกัน

Feature ต่างๆของ Web Application มีดังนี้

Login/Register

  • ฟีเจอร์ในการสมัครใข้งาน/ ล็อกอิน (การล็อกอินจะใช้ Username ที่สมัครไว้ ไม่ใช้อีเมล)

Basic Navigation

  • โฮมเพจ : หน้า Landing page
  • เลือกห้อง : เมื่อคลิกหน้านี้จะเข้าไปในหน้าที่ทำการเลือกห้อง (เลือกแชแนล)
  • สร้างห้อง : เมื่อคลิกที่หน้านี้ ผู้ใช้งานจะสามารถสร้างห้องของตัวเองได้
  • แก้ไขข้อมูลส่วนตัว : ผู้ใช้งานสามารถแก้ไขชื่อจริง นามสกุล หรือข้อมูลอื่นๆของตนเองได้ ยกเว้น Username กับ Password ที่จะต้องขออนุมัติทาง admin

Main Function

  • การสร้างห้อง : ผู้ใช้งานสามารถสร้างห้องและใส่รายละเอียดคร่าวๆลงไปได้ อย่างเช่น ข้อมูลภารกิจ เพื่อให้ผู้ใช้งานท่านอื่นเห็นและเข้าใจคร่าวๆเกี่ยวกับสิ่งที่วางแผน
  • การลบห้อง : มีเพียงผู้สร้างห้องเท่านั้น ที่สามารถลบห้องของตนเองได้
  • การวางแผน (การวาดภาพ) : มีเพียงผู้ที่สร้างห้องเท่านั้น ที่สามารถวางแผนในห้องของตนเองได้ ผู้ใช้งานท่านอื่นไม่สามารถแก้ไขการวางแผนของเจ้าของห้องได้
  • การระบุพิกัดตนเองและผู้ใช้งานท่านอื่น : ผู้ที่อยู่ในห้องเดียวกัน จะสามารถเห็นพิกัดของกันและกันได้

หมายเหตุ: พิกัดที่รับส่ง จะมีค่า Error อยู่ที่ 5 - 15 เมตร แล้วแต่ความแรงของสัญญาน Internet

ข้อมูลทางเทคนิค

  • Backend เขียนด้วย Django
  • Database ในการเก็บภาพที่วาด ใช้ Postgresql และ postgis ในการเก็บข้อมูล
  • Database ที่ใช้ในการแสดงผลตำแหน่งพิกัดผู้ใช้ เป็น Redis
  • แผนที่ ใช้ Leaflet ซึ่งเป็น Open Sources
  • การ Broadcast สัญญาน ใช้ Django channels ซึ่งทำงานร่วมกับ Websocket ช่วยให้ทุกคนเห็นพิกัดตนเองได้แบบ Realtime หรือ Near Real Time
